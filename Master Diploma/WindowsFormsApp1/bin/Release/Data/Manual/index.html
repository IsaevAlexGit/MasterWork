1. Вход в программу, выбор локализации карты, цвета интерфейса

При входе в приложение пользователь видит окно входа, где ему необходимо выбрать локализацию карты, то есть локализацию для подписей названия улиц, городов и стран на карте. В приложении доступны две локализации: русская и английская. Выбирается локализация в выпадающем списке.

Также на этом окне можно выбрать цвет интерфейса. По умолчанию цвет интерфейса – зеленый. В модальном окне пользователь может выбрать любой цвет для интерфейса, кроме черного цвета. Чтобы открыть модальное окно, надо нажать кнопку в левом нижнем углу «Выбрать цвет интерфейса».

В правом нижнем углу окна входа размещена веб-инструкция для пользователей, чтобы открыть её надо нажать кнопку «Руководство пользователя».

Чтобы перейти на следующее окно, необходимо нажать кнопку «Продолжить». Следующее окно откроется только в случае успешного подключения к интернету. Статус подключения можно посмотреть в левом верхнем углу окна.

Стартовое окно представлено на рисунке 1.

2. Начальная загрузка пользовательских файлов

С окна входа пользователь переходит на окно начальных настроек. Здесь пользователь загружает данные, с которыми будет взаимодействовать геоинформационная система «Оптимум».

В левом верхнем углу пользователь загружает значок объекта социальной инфраструктуры, для этого надо нажать по кнопке «Загрузить значок».

Загружаемый файл должен:

  • иметь png-расширение;
  • иметь разрешение больше 5х5 пикселов и меньше 90х90 пикселов.

Загруженный значок будет отображен слева от кнопки «Загрузить значок». По умолчанию для отображения объектов социальной инфраструктуры на карте используется значок синего восьмиугольника. При попытке загрузить значок, не удовлетворяющий данным условиям, приложение сообщит пользователю о конкретной ошибке.

На рисунке 2 представлен элемент окна начальных настроек загрузки значка.


В правом верхнем углу пользователь задает тип социальной инфраструктуры или название бизнеса – объекта, с которым он будет работать далее. Название вводится в текстовое поле и должно содержать от 2 до 20 символов. Если пользователь задаст некорректные данные, приложение сообщит ему о конкретной ошибке. На рисунке 3 представлен элемент окна начальных настроек ввода названия объекта инфраструктуры.


В левой части экрана пользователь задает частные критерии оптимальности, на которых базируется поиск оптимальных точек для открытия нового бизнеса или объекта социальной инфраструктуры.

В выпадающем списке количества критериев пользователь выбирает число от 1 до 7, на основании скольких критериев будет производиться поиск оптимумов. Для каждого частного критерия необходимо задать три параметра: название критерия, направление и весовой коэффициент.

Название критерия задается в текстовое поле критерия, название должно быть от 2 до 30 символов. Названия не должны быть одинаковыми. Нельзя задать критерий с названием «Скрыть».

Направление критерия задаётся в выпадающем списке. Направление критерия отвечает за предпочтение максимизации или минимизации данного критерия при поиске оптимальных точек.

Весовой коэффициент критерия задается также в текстовое поле. Весовой коэффициент отвечает за важность данного критерия, на сколько важен данный критерий при поиске оптимальных точек. Значения весовых коэффициентов должны быть числами в диапазоне [0; 1]. Чем ближе весовой коэффициент к единице, тем важнее данный критерий для пользователя. Сумма всех весов должна быть равна единице. На рисунке 4 представлен элемент окна начальных настроек ввода частных критериев оптимальности.


После успешной установки частных критериев пользователь загружает пространственные файлы: зону анализа (региона), полигональные объекты на карте с пользовательской информацией, объекты социальной инфраструктуры. На рисунке 5 представлены кнопки, по нажатию на которые пользователь загружает соответствующие файлы.


Файл с граничными точками зоны анализа представляет из себя CSV-файл, в котором хранится перечень граничных точек рассматриваемого региона. При загрузке файла необходимо учитывать, что данный файл должен:

  • загружаемый файл не должен быть открыт на компьютере во время загрузки;
  • данный файл должен существовать;
  • файл должен иметь расширение .csv;
  • файл должен иметь не пустое название;
  • файл должен быть доступен для чтения;
  • файл должен быть меньше 1 Мб;
  • файл должен иметь разделитель – запятая;
  • файл должен быть не пустым и содержать информацию;
  • координаты граничных точек зоны анализа должно быть больше 3.

Структура файла: широта и долгота точки.

3. Зуммирование (масштабирование) карты

Зуммирование – это приближение и отдаление карты. Чтобы приблизить карту надо прокрутить колесо мыши вверх, чтобы отдалить карту необходимо прокрутить колесо мыши вниз.

4. Перетаскивание карты

Для того, чтобы пользователь мог перетаскивать карту необходимо зажать левую кнопкой мыши на карте и потянуть карту.

5. Отображение всплывающих подсказок

При наведении на некоторые элементы интерфейса пользователь может увидеть всплывающие подсказки, которые уточняют или поясняют назначение данного элемента интерфейса (кнопки, текстового поля, переключателя). Всплывающая подсказка отображается пять секунд.

6. Элементы главного окна

После успешной загрузки файлов пользователь попадает на главное окно, которое представлено на рисунке 6.

В верхней части главного окна располагается меню (рисунок 7).

В левой части главного окна располагается левая панель, которую пользователь может спрятать, нажав на стрелочку «Скрыть боковую панель» (рисунок 8).

Справа вверху отображаются условные обозначения карты, где в том числе отображается значок объекта социальной инфраструктуры, загруженный ранее пользователем (рисунок 9).

7. Отображение объектов социальной инфраструктуры

В верхней части левой панели пользователь может включать и выключать отображение всех загруженных им объектов социальной инфраструктуры. Для этого надо нажать на переключатель «Отобразить», после чего на карте появятся все объекты социальной инфраструктуры (рисунок 10). Переключатель «Скрыть» выключит отображение объектов на карте.

8. Изменение выводимой информации об объекте инфраструктуры при наведении на маркер

Для того, чтобы изменить информацию об объекте инфраструктуры, которая выводится при наведении на маркер, необходимо в левой панели в блоке «Всплывающая подсказка» установить соответствующее поле.

При установке поля «Название объекта» при наведении на маркер будет отображаться название объекта инфраструктуры, заданное в начале пользователем (рисунок 11).

При установке поля «Все данные» при наведении на маркер будут отображаться все данные, которые были указаны в файле пользователем (рисунок 12).

Если загруженный файл с информацией об объектах инфраструктуры не хранил никакой информации, кроме идентификатора и координат, то при наведении на маркер будет написано «Данные отсутствуют».

9. Просмотр информации об объекте социальной инфраструктуры

Для того, чтобы просмотреть подробную информацию об объекте социальной инфраструктуры, необходимо щёлкнуть один раз левой кнопкой мыши по маркеру, после чего откроется окно с информацией об этом объекте (рисунок 13).

Если среди указанной информации есть веб-ссылка, то в открывшемся окне с информацией об объекте инфраструктуры пользователь может перейти на указанный сайт, щелкнув по полю с сайтом дважды левой кнопкой мыши.

Если загруженный файл с информацией об объектах инфраструктуры не хранил никакой информации, кроме идентификатора и координаты, то при клике на маркер объекта инфраструктуры пользователю будет показано сообщение: «Данные отсутствуют» (рисунок 14).

10. Отображение зоны анализа

Для того, чтобы отобразить на карте зону анализа, необходимо не левой панели главного окна в блоке «Зона анализа» выбрать переключатель «Отобразить» (рисунок 15).

При установке переключателя «Скрыть» на карте не будет отображаться зона анализа (рисунок 16).

11. Площадная раскраска

Все загруженные полигональные объекты можно отображать на карте в виде площадной раскраски, то есть цветовой визуализации полигонов, насыщенность цвета которых зависит от значения выбранного критерия относительно других полигонов. Чем больше значение выбранного критерия для раскраски, тем более насыщенным будет цвет раскраски данного полигона.

На левой панели в выпадающем списке можно выбрать любой из загруженных ранее критериев, для которого будет построена площадная раскраска (рисунок 17-19). Если выбрать пункт «Скрыть», то площадная раскраска будет скрыта с карты (рисунок 20).

Как только пользователь включает отображение на карте любой из площадных раскрасок в нижней части главного окна появляется нижняя панель, которую также можно спрятать, опустив её на кнопку «Скрыть нижнюю панель». Нижняя панель позволяет визуализировать отображающуюся в данный момент площадную раскраску на карте (рисунок 21).

12. Настройка площадной раскраски

Пользователь может настраивать раскраску полигонов: изменять цвет раскраски, изменять прозрачность заливки полигонов, изменять толщину границ у полигонов, а также задавать количество оттенков от 1 до 10, в которое будут окрашены полигоны.

Инструменты, которые позволяют это сделать расположены на нижней панели главного окна. По умолчанию данные элементы интерфейса скрыты от пользователя. Они становятся видимыми, когда пользователь раскрашивает карту по одному из критериев.


Чтобы изменить цвет раскраски, необходимо на нижней панели щелкнуть левой кнопкой мыши по панели «Легенда карты», далее в диалоговом окне выбрать цвет для раскраски полигонов, после чего в данном окне нажать кнопку «Ок» (рисунок 22). На рисунке 23 представлена карта с раскрашенными в заданный цвет полигонами.


Чтобы изменить прозрачность раскраски, необходимо на нижней панели передвигать ползунок «Прозрачность раскраски». Самое левое положение ползунка – раскраска прозрачна на 100 %, на карте отображены только границы полигонов. Самое правое положение ползунка – раскраска самая яркая. На рисунке 24 представлена раскраска полигонов с положением ползунка в левой части. На рисунке 25 представлена раскраска полигонов с положением ползунка по середине. На рисунке 26 представлена раскраска полигонов с положением ползунка в правой части.


Чтобы изменить толщину границ полигонов, необходимо на нижней панели передвигать ползунок «Толщина границ полигонов». Самое левое положение ползунка – полигоны не имеют границ (рисунок 27). Самое правое положение ползунка – полигоны имеют максимальную толщину границ (рисунок 28).


Чтобы изменить количество оттенков раскраски полигонов, необходимо на нижней панели передвигать ползунок «Количество оттенков раскраски». Самое левое положение ползунка – полигоны раскрашены в один оттенок (рисунок 29). Самое правое положение ползунка – полигоны раскрашены в десять оттенков (рисунок 30). Заданное количество оттенков отображается также в легенде карты на нижней панели. Количество оттенков раскраски также влияет на оттенки для точечной раскраски.

13. Точечная раскраска

Также полигоны могут быть визуализированы точечной раскраской. На левой панели в выпадающем списке можно выбрать любой из загруженных ранее критериев, для которого будет отображена точечная раскраска (рисунок 31-33). Если выбрать пункт «Скрыть», то точечная раскраска будет скрыта с карты. Точечная раскраска может быть отображена поверх площадной раскраски (рисунок 34). Изменяя положение ползунка «Количество оттенков раскраски», пользователь будет также изменять количество оттенков для точечной раскраски.

14. Установка точек-кандидатов на карте

Для того, чтобы установить на карте точку-кандидат пользователю необходимо щелкнуть дважды левой кнопкой мыши по карте. Если данная точка-кандидат удовлетворяет следующим условиям, то она будет установлена, а около неё будет отображена буферная зона красного цвета (рисунок 35):

  • Точка-кандидат находится в пределах загруженной зоны анализа. В противном случае пользователь получит предупреждение.
  • Точка-кандидат принадлежит одному из полигонов. В противном случае пользователь должен подтвердить установку точки-кандидата.
  • Сумма установленных точек-кандидатов не должна превышать 30 штук. В противном случае пользователь получит предупреждение.
  • Устанавливаемая точка-кандидат должна быть размещена дальше 25 метров относительно любой другой установленной точки-кандидата.

15. Удаление точек-кандидатов с карты

Для того, чтобы убрать установленную ранее на карте точку-кандидат необходимо щелкнуть правой кнопкой мыши по данной точке, после чего она пропадет с карты (рисунок 36). Таким же образом с карты удаляются оптимальные точки.

16. Установка радиуса буферной зоны

Для того, чтобы задать радиус буферной зоны, в которой будет происходить подсчет значений частных критериев всех полигонов, центры которых попали в эту буферную зону, необходимо в меню нажать на кнопку «Задать радиус». После чего пользователь перейдет на окно «Установка радиуса поиска» (рисунок 37). На данном экране пользователь задает радиус буферной зоны (в метрах) путём прокрутки ползунка. По умолчанию радиус равен 500 метрам. Минимальный радиус поиска может быть равен 300 метрам. Максимальное значение радиуса зависит от площади загруженной зоны анализа и вычисляется динамически. Если радиус был изменён, и на карте были установлены точки-кандидаты, то после закрытия этого окна буферные зоны будут отрисованы заново с новым радиусом (рисунок 38-39).

17. Поиск оптимальной точки на карте

Для того, чтобы найти оптимальные точки на карте для размещения объекта социальной инфраструктуры или бизнеса, необходимо для начала разместить на карте больше двух точек-кандидатов, после чего в меню на главном окне нажать на кнопку «Найти оптимум». После этого на карте будут отображены все оптимальные точки, которые были найдены математическим алгоритмом многокритериальной оптимизации: методами свертывания критериев.

Точки-кандидаты, которые выбрали какие-либо методы оптимальными будут отображены на карте зеленым цветом. Точки-кандидаты, не выбранные никакими методами, также останутся красным цветом. Точка-кандидат, которая оказалась одной из оптимальных и при этом получила большинство голосов будет отображена темно-зелёным цветом с желтым контуром буферной зоны.

Результат поиска оптимальных точек представлен на рисунке 40.

При наведении на оптимальную точку будет отображен список методов, которые выбрали эту точку оптимальной (рисунок 41-42).

18. Удаление всех точек-кандидатов

Чтобы быстро удалить с карты все точки-кандидаты, необходимо в меню нажать на кнопку «Удалить кандидатов». Тогда с карты пропадут все точки-кандидаты. Если были найдены оптимальные точки, то они останутся на карте (рисунок 43-44).

При наведении на оптимальную точку будет отображен список методов, которые выбрали эту точку оптимальной (рисунок 41-42).

19. Сохранение оптимальной точки в файл

Для того, чтобы сохранить данные об оптимальных точках в файл, пользователю необходимо нажать на кнопку «Сохранить оптимумы» в меню. Если на эту кнопку нажать, когда на карте не найдены оптимальные точки, пользователь получит предупреждение. Если оптимальные точки найдены пользователь выбирает файл для сохранения, который должен удовлетворять следующим условиям:

  • загружаемый файл не должен быть открыт на компьютере во время загрузки;
  • данный файл должен существовать;
  • файл должен иметь расширение .csv;
  • файл должен иметь не пустое название;
  • файл должен быть открыт для записи;
  • пользователь должен дать согласие, что все данные хранящиеся в данном файле будут перезаписаны;

Если одно из условий, описанных выше не выполняется, пользователь получит сообщение о конкретной ошибке, по которой приложение не может выполнить запись данных. Если все условия соблюдены, то откроется диалоговое окно, в котором пользователь выбирает файл для сохранения данных (рисунок 45).

В созданном файле хранится информация об оптимальных точках: координата оптимальной точки, радиус буферной зоны поиска, значения всех критериев в данной буферной зоне, перечень всех методов, которые выбрали эту точку оптимальной (рисунок 46).

20. Просмотр оптимальных точек в веб-картографических сервисах

После того, как пользователь нашел оптимальные точки, он может открыть их в браузере и просмотреть в одной из известных веб-картографических систем: Google, Яндекс, 2ГИС и OpenStreetMap. Для этого в меню надо выбрать пункт «Оптимум в браузере» и в выпадающем списке выбрать сервис (рисунок 47).

21. Изменение пользовательских файлов и данных

В любой момент работы с приложением пользователь может вызвать окно для изменения данных, которое расположено первым пунктом в меню на главном окне. Данный экран позволяет в любой момент изменить значок объекта социальной инфраструктуры, изменить название объекта, количество и значение частных критериев оптимальности, загрузить новые файлы с зоной анализа, полигональными объектами или объектами инфраструктуры и бизнеса (рисунок 48).

22. Центрирование карты

В любой момент работы с приложением пользователь может отцентрировать карту на загруженной зоне анализа. Для этого в пункте меню на главном окне необходимо кликнуть по пункту «Центрирование карты», после чего загруженная зона анализа появится в центре карты (рисунок 49).

23. Экспорт текущего положения карты

Для того, чтобы сохранить текущее положение карты со всеми отображёнными на ней пространственными объектами, необходимо в меню нажать на кнопку «Сохранить карту». Далее выбрать название файла и путь, по которому сохранится данная фотография и подтвердить свой выбор (рисунок 50). После чего изображение карты будет сохранено.

Также для открытия данного диалога сохранения карты можно нажать сочетание кнопок «Ctrl+S», находясь на главном окне.

24. Открытие руководства пользователя

В меню на главном окне, на окне входа и на окне поиска нормы на душу населения можно открыть веб-инструкцию для приложения. Для этого необходимо щелкнуть в меню на главном окне по пункту «Руководство».

26. Автоматический поиск оптимальных точек

Первый алгоритм поиска оптимальных точек был рассмотрен раньше. Пользователь устанавливает на карте несколько точек-кандидатов, и из этих точек-кандидатов находятся оптимальные точки. Второй вариант предполагает автоматический поиск оптимальных точек. Для этого пользователю надо только загрузить все требуемые файлы при входе. Данный вариант самостоятельно сканирует всю загруженную зону анализа и находит требуемое количество оптимальных точек от 1 до 10.

На левой панели пользователь в выпадающем списке указывает число от 1 до 10, сколько оптимальных точек отобразить ему (рисунок 61). По умолчанию в выпадающем списке задано 10.

Далее в меню необходимо нажать по кнопке «Автопоиск» и в меню пункта «Автопоиск» нажать «Найти» (рисунок 62). После чего на карте отобразится требуемое количество оптимальных точек, найденных автоматически (рисунок 63).

На автопоиск также влияет радиус буферной зоны, частные критерии оптимальности, загруженные файлы о пространственных данных. Изменив эти данные, можно будет получить новые оптимальные точки (рисунок 64).

Если нажать в «Автопоиск» пункт «Скрыть», то отображение автопоиска будет скрыто, если выбрать «Отобразить», то точки вновь будут отображены. Пункт «Удалить» полностью удаляет результаты автопоиска.

27. Поиск нехватки или избыточности объектов инфраструктуры на душу населения

В геоинформационной системе «Оптимум» также представлена рекомендательная система, которая позволяет выявить нехватку и/или избыточность объектов социальной инфраструктуры при указанном значении нормы на душу населения. И при их нехватке «Оптимум» отобразит на карте недостающее до нормы количество оптимальных точек.

Если пользователь хочет воспользоваться рекомендательной системой по поиску нехватки или избыточности объектов инфраструктуры на норму на душу населения, то ему необходимо загрузить файл с полигонами, удовлетворяющий следующим требованиям:

  • каждый полигон должен иметь два или более частных критерия;
  • первым значением в критериях должно быть указано количество жителей в этом полигоне;
  • вторым значением в критериях должно быть указано количество данных объектов инфраструктуры в этом полигоне, остальные пять критериев могут быть использованы для другой пользовательской информации.

Чтобы воспользоваться данным сервисом, необходимо в меню на главном окне нажать на пункт «Норма на душу населения» (рисунок 65). После чего пользовательский файл с полигонами будет проверен на соответствие требованиям. Если файл с полигонами является корректным, откроется окно «Поиск нормы на душу населения» (рисунок 66).

На данном окне пользователь может также включать и отключать отображение на карте объектов социальной инфраструктуры (рисунок 67).

Пользователь может также задать радиус для буферных зон.

Под радиусом поиска указываются поля «Норма объектов» и «На душу населения». По умолчанию там уже указаны значения: 1 объект инфраструктуры на 1000 жителей.

Значение «Нормы объектов» должно быть меньше значения «На душу населения», иначе пользователь получит предупреждение от системы о вводе некорректных данных.

Далее необходимо нажать кнопку «Анализ объектов», после чего система анализирует каждый полигон зоны анализа, подсчитывает общее количество жителей в зоне анализа, общее количество объектов инфраструктуры в зоне анализа, делает вывод о том, хватает или нет данных объектов инфраструктуры в зоне анализа.

Если объектов меньше, чем нужно, то приложение отобразит на карте недостающее количество объектов инфраструктуры в наилучших для этого местах (рисунок 68).

Если объектов больше, чем нужно, то приложение предупредит об этом пользователя, но при его желании может отобразить 10 наилучших оптимальных точек для открытия такого объекта инфраструктуры (рисунок 69).

Найденные точки нормы можно отобразить и скрыть с карты, также на этом окне можно ознакомиться с руководством пользователя и сохранить карту в виде изображения.
